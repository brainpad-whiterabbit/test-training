# 演習の目的

この演習では、電卓Webアプリの開発を通じて、要件定義からテスト、CI/CDまでを早い段階で確認するシフトレフトの進め方を学びます。

  <p class="training-kicker">Shift-left training</p>
  <h2>品質を、最後の確認から日々の設計判断へ。</h2>
  <p>
    小さな電卓Webアプリを題材に、レビュー、テスト分析、CI/CD、ユーザビリティ確認をスプリントの流れに組み込みます。
  </p>

## 学習内容

- 要件とテスト観点を早期にすり合わせる
- 設計レビューで品質リスクを見つける
- 実装とテストを継続的に確認する
