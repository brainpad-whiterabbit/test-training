## 学習内容

## アジャイル開発とスプリント

アジャイル開発は、短いサイクルで開発と改善を繰り返しながら価値を提供する開発手法です。
その短い開発サイクルを「スプリント」と呼び、通常は1〜4週間程度の期間で、要件定義・設計・実装・テストを繰り返します。

---

## シフトレフトとは

シフトレフトとは、品質確認やテスト活動を開発の後半ではなく、より早い工程へ移動して実施する考え方です。

従来型ではテスト工程に作業が集中しやすく、問題発見が遅れる傾向があります。
一方、シフトレフトでは開発工程と並行してテスト活動を進めることで、早期フィードバックと品質向上を実現します。

---

シフトレフトありなしのテスト工程の違いの例

| スプリント期間 | 開発工程     | テスト工程          | テスト工程 (シフトレフトあり) |
| ------- | -------- | --------------------------- | ---------------- |
| 前半      | 要件定義 |                             | 要件レビュー、テスト分析     |
| 前半〜中盤   | 設計 |                             | 設計レビュー、テスト設計     |
| 中盤      | 実装 |                             | テスト実装            |
| 後半      | テスト | テスト分析、テスト設計、テスト実施（探索的テスト含む） | 探索的テスト           |

---

## シフトレフトを支えるCI/CD

シフトレフトを実現するための重要な技術として、CI/CDがあります。

* CI（Continuous Integration）
    * コード変更を継続的に統合する仕組み
* CD（Continuous Delivery / Deployment）
    * 継続的にデプロイ可能な状態を維持する仕組み

CI/CD環境では、開発中に以下を自動的に確認できます。

* 自動テストの実行
* ビルド確認
* デプロイ可能かの確認

その結果、問題が発生した場合でも即座にフィードバックされ、開発者は素早く修正しながら開発を進められます。

---

## まとめ

シフトレフトでは、テスト活動を開発の後半にまとめるのではなく、開発工程と並行して実施します。
さらに、CI/CDによって継続的に品質確認を行うことで、迅速なフィードバックと品質向上を実現できます。

本取り組みでは、

* シフトレフトに沿ったテスト工程
* CI/CDを活用した開発

を実際に経験してもらうことで、シフトレフト開発の考え方を学習してもらいます。
