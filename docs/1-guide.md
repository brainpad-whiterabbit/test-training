# 演習の進め方

---

## 現状確認

まずは電卓Webアプリを立ち上げ、現在の挙動や画面デザインを確認してください。
起動方法については、READMEを参照してください。

実際に操作してみると、以下のような課題があることが分かると思います。

- レスポンスが遅い
- 一部の機能が未実装である
- ボタンの色がすべて同じで、視認性が低い

本演習では、スプリント形式で改善を繰り返しながら、これらの課題を少しずつ解消していきます。
最終的には、より使いやすく完成度の高い電卓Webアプリに仕上げることを目指します。

## 演習概要

本演習では、1スプリントごとに基本的に1機能を追加しながら、複数回スプリントを回してもらいます。

以下のスプリントに対応するスプリントページが用意されているので、これらのページを参照しながら演習を進めてください。

- チュートリアルスプリント
- 演習スプリント

チュートリアルスプリントは、講師と一緒に作業をすすめてもらい、スプリントの進め方や各工程で実施する内容を理解してもらいます。
演習スプリントは、チュートリアルスプリントで理解した内容をもとに、1人でスプリントを回してもらいます。

---

## スプリント内で実施する内容

要件定義と設計については、事前に用意されているドキュメントページを参照してください。
演習では、要件定義と設計以外の活動を実施してもらいます。

| スプリント期間 | 開発工程           | テスト工程               |
| -------------- | ------------------ | ------------------------ |
| 前半           | 要件定義(作成済み) | 要件レビュー・テスト分析 |
| 前半〜中盤     | 設計(作成済み)     | 設計レビュー・テスト設計 |
| 中盤           | 実装               | テスト実装               |
| 後半           | テスト             | 探索的テスト             |
