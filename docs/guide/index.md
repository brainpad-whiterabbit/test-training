# 演習の進め方

各スプリントでは、要件確認、設計、実装、テスト、ふりかえりを短いサイクルで実施します。

## 基本の流れ

1. 要件を確認する
2. 品質リスクを洗い出す
3. 実装とテストを進める
4. 成果をレビューする
5. ふりかえりで学びを整理する

## 作成する成果物

- 要件レビュー結果
- テスト分析メモ
- 設計レビュー結果
- テストケース
- ふりかえりメモ

## レビュー観点

レビューでは、仕様の抜け漏れ、ユーザー影響、テスト可能性、保守性を確認します。

- 要件は具体的に判断できるか
- 例外や境界値が考慮されているか
- 実装方針がテストしやすいか

## CI/CDの確認

CI/CDでは、テスト、lint、型チェックを自動実行し、変更の品質を継続的に確認します。

```bash
./scripts/check.sh
```
