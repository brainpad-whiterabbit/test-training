# 要件定義・設計書

演習用に作成した電卓Webアプリの簡易的な要件定義・設計書です。

## 1. システム概要

### 1.1 システム名

電卓Webアプリ

### 1.2 目的

利用者がブラウザ上で四則演算を簡単に行えるWebアプリを提供する。

### 1.3 対象利用者

- ブラウザ上で簡単な計算を行いたい一般利用者

---

## 4. 非機能要件

|     No | 分類             | 要件内容                                 | 基準                                   | レビュー反映(基準) | 対応スプリント |
| -----: | ---------------- | ---------------------------------------- | -------------------------------------- | ------------------ | -------------- |
| NF-001 | 性能             | ボタン押下後、画面表示を速やかに更新する | 0.5秒以内                              |                    |                |
| NF-002 | 対応ブラウザ     | 主要ブラウザで利用できる                 | Chrome、Edge、Safariの最新版           |                    |                |
| NF-003 | レスポンシブ対応 | PCで画面崩れなく利用できる               | 幅375px以上で操作可能                  |                    |                |
| NF-004 | 操作性           | 主要ボタンを視認しやすく配置する         | 数字・演算子・結果欄が明確に区別できる |                    |                |
| NF-005 | 可用性           | ページ再読み込み後もアプリを再利用できる | 再読み込み後に初期状態で表示される     |                    |                |
| NF-006 | セキュリティ     | 入力値としてスクリプトを実行しない       | 画面入力は数字・演算子のみに制限する   |                    |                |
| NF-007 | 保守性           | 機能ごとに処理を分割し修正しやすくする   | 計算処理・画面表示処理を分離する       |                    |                |

| 乗算               | 初期状態                 | `3 × 4 =` を押下する          | `12` が表示される               |                        |                |

| 小数計算           | 初期状態                 | `1.5 + 2.5 =` を押下する      | `4` が表示される                |                        |                |
