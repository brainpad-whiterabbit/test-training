# 要件定義

演習用に作成した電卓Webアプリの簡易的な要件定義書です。

## 1. システム概要

### 1.1 システム名

電卓Webアプリ

```text
+------------------------+
| 電卓Webアプリ           |
|    (式表示エリア)  12 + |
+------------------------+
|            (表示エリア) |
|                     0  |
+------------------------+
| CE | C  | ±  | %  | ÷  |
| 7  | 8  | 9  | x  | -  |
| 4  | 5  | 6  | +  | =  |
| 1  | 2  | 3  | 0  | .  |
+------------------------+
```

### 1.2 目的

ユーザがブラウザ上で基本的な電卓機能を行えるWebアプリを提供する。

### 1.3 対象利用者

ブラウザ上で簡単な計算を行いたいユーザ

---

## 3. 機能要件

機能要件として、システムの機能・振る舞いを定義する。

|   No | 分類   | 機能名           | 要件内容                                                                                               | レビュー点 | 対応スプリント           |
| ---: | ------ | ---------------- | ------------------------------------------------------------------------------------------------------ | ---------- | ------------------------ |
| F-01 | 入力   | 数字入力         | 数字ボタン（0〜9）押下時に、押下された数値を入力欄へ追加し、入力内容を表示する。                       |            | 対応済み                 |
| F-02 | 入力   | 小数点入力       | 「.」ボタン押下時に、現在入力中の数値へ小数点を追加し、入力内容を表示する。                            |            | 対応済み                 |
| F-03 | 入力   | 符号反転         | 「±」ボタン押下時に、現在入力中の数値の符号を反転し、反転後の値を表示する。                            |            | 対応済み                 |
| F-04 | 入力   | 入力制御         | 演算子入力の後に別の演算子が入力された場合に、後続の演算子入力を受け付けず、直前の入力内容を保持する。 |            | 対応済み                 |
| F-05 | 入力   | 計算実行         | 「=」ボタン押下時に、入力された計算式を演算し、計算結果を表示する。                                    |            | 対応済み                 |
| F-07 | 演算   | パーセント計算   | 「%」ボタン押下時に、現在入力中の数値を百分率として演算し、計算結果を表示する。                        |            | 対応済み                 |
| F-06 | 演算   | 加算             | 入力された2つの数値を加算(+)し、計算結果を表示する。                                                   |            | 対応済み                 |
| F-06 | 演算   | 減算             | 入力された2つの数値を減算(-)し、計算結果を表示する。                                                   |            | チュートリアルスプリント |
| F-06 | 演算   | 乗算             | 入力された2つの数値を乗算(x)し、計算結果を表示する。                                                   |            | 演習スプリント1          |
| F-06 | 演算   | 除算             | 入力された2つの数値を除算(/)し、計算結果を表示する。                                                   |            | 演習スプリント1          |
| F-08 | エラー | エラー表示       | 0除算など計算不能な演算が実行された場合に、エラーメッセージ(Error)を表示する。                         |            | 演習スプリント1          |
| F-08 | 制限   | 表示桁数制限     | 最大12桁(整数部最大10桁、小数部最大8桁)まで入力を受け付ける。上限を超えた場合 、表示の値を維持する。   |            | 演習スプリント2          |
| F-08 | 制限   | 計算結果桁数制限 | 演算結果が表示桁数を超えた場合に、エラーメッセージ(Overflow)を表示する。                               |            | 演習スプリント2          |
| F-09 | クリア | 初期化           | 「C」ボタン押下時に、入力1・入力2を含む入力中の計算式、計算結果、演算状態を初期化し、入力欄に `0` を表示する。 |            | 対応済み                 |
| F-10 | クリア | 入力値リセット   | 「CE」ボタン押下時に、現在入力中の演算対象値を `0` にリセットする。入力2の入力後に押下した場合、入力1と演算子は保持する。 |            | 演習スプリント3          |

## 4. 非機能要件

非機能要件として、機能以外でどのような品質・制約で動作するか定義する。
(演習用に抜粋)

|    No | 分類         | 要件内容                                                                   | レビュー点 | 対応スプリント  |
| ----: | ------------ | -------------------------------------------------------------------------- | ---------- | --------------- |
| NF-01 | 性能         | ボタン押下後1秒以内に画面表示を更新する                                    |            | 対応済み        |
| NF-02 | 対応ブラウザ | Google Chromeで利用できる                                                  |            | 対応済み        |
| NF-03 | 操作性       | 数字(青)・演算子(オレンジ)・クリア系ボタン(赤)が色分けされており識別できる |            | 演習スプリント5 |
| NF-04 | 可用性       | ページ再読み込み後に初期状態で操作を開始できる                             |            | 対応済み        |
