# チュートリアルスプリント

## スプリント概要

現状、加算の機能のみ実装されています。四則演算の残りの機能についてこのスプリントで実装してもらいます。

具体的に

- 減算
- 乗算
- 除算

の機能を追加することになります。

## 全体の流れ

再掲ですが、スプリント内で実施する内容は以下の通りです。

| スプリント期間 | 開発工程     | テスト工程                   |
| -------------- | ------------ | ---------------------------- |
| 前半           | ~~要件定義~~ | 要件レビュー, ~~テスト分析~~ |
| 前半〜中盤     | ~~設計~~     | ~~設計レビュー~~, テスト設計 |
| 中盤           | 実装         | テスト実装                   |
| 後半           | ~~テスト~~   | 探索的テスト                 |

以下の順番で実施してもらいます。

1. 要件レビュー
1. テスト設計
1. 実装
1. テスト実装
1. 探索的テスト

## 要件レビュー

今回は単体の機能を追加を目指す1スプリント内の要件レビューなため、本演習では以下と簡易的に定義します。

「スプリントで追加する機能の内容について、実現可能性、完全性、一貫性、明確性、テスト可能性などを確認する活動」

具体的に、追加機能の要件の記述に曖昧さや矛盾、不足がないか等の確認と、受入条件との整合性およびテスト可能性を確認してもらいます。

### レビュー対象を確認

まず、[要件定義書](97-requirements.md)に記載されている内容を確認してください。機能要件、非機能要件、受入条件に具体的な要件が記載されて定義されています。

- [機能要件](97-requirements/#3)
- [非機能要件](97-requirements/#4)
- [受入条件](97-requirements/#5)

各項目の表のうち「対応スプリント」列が「チュートリアルスプリント」となっている行の記述がこのスプリントで追加する機能に対応する要件です。

### レビューを実施

[レビュー観点](98-viewpoints.md)に従って対象の項目をレビューして、もし修正点があれば、レビュー点の列に修正内容を記載してください。

1. リポジトリのMarkdownファイルを開く(`docs/97-requirements.md`)
1. レビュー内容を記載
    - 修正点があれば修正を反映して、メモ程度でよいのでレビュー点を記載

    ```markdown title="docs/97-requirements.md"
    |    No | 分類 | 機能名   | 要件内容                       | レビュー点             | 対応スプリント           |
    | ----: | ---- | -------- | ------------------------------ | ---------------------- | ------------------------ |
    | F-006 | 演算 | 四則演算 | 加算、減算、乗算、除算ができる | {ここにレビュ点を記載} | チュートリアルスプリント |
    ```

1. 変更をコミット

    ```bash title="コミット例"
    git add docs/97-requirements.md
    git commit -m "docs: チュートリアルスプリントの要件レビュー内容を反映"

    ```

1. 変更をプッシュ

    ```bash
    git push origin <your-branch-name>
    ```

## テスト設計

テスト観点に基づき、テストケースを設計します。

### 関連するテスト観点を抽出

[テスト観点](98-viewpoints.md)に記載されているテスト観点を確認してください。

テスト観点は要件が増える度に見直され追加されます。現在はチュートリアルスプリントを実施しているので、対応スプリントが「チュートリアルスプリント」「対応済み」となっている行のテスト観点から必要なもの(減算、乗算、除算に関連)を抽出してください。(特に書き出す必要はないです)

- 抽出例
    - TV-015 減算
        - 減算を対象としたテスト観点なため当然入る
    - TV-018 小数演算
        - 小数を含んだ四則演算を対象としたテスト観点なため入る

### テストケース作成

抽出したテスト観点を踏まえて、テストケースを作成してください。以下の表を埋める形でもよいですし、形式を変えてもよいです。

| テスト観点 | テストケース | 入力 | 期待値 |
| ---------- | ------------ | ---- | ------ |
|            |              |      |        |
|            |              |      |        |
|            |              |      |        |

#### テストケース作成例

参考にテストケースの作成例 (TV-015 減算のみ)を示します。

- 入力値を同値分割
    - 最小値, 負数, 0, 正数, 最大値
- 右と左の入力値の組み合わせを同値分割
    - (負 - 負)
    - (負 - 正)
    - (正 - 負)
    - (正 - 正)
    - 0を含む
    - 最大値を含む
    - 最小値を含む
    - (最大値 - 最大値)
    - (最小値 - 最小値)

テストケースに落とし込むと以下のようになります。

| テスト観点 | テストケース             | 入力          | 期待値   |
| ---------- | ------------------------ | ------------- | -------- |
| 減算       | 負数同士の減算ができる   | -1 - (-1)     | 0        |
| 減算       | 負数から正数を減算できる | -1 - 1        | -2       |
| 減算       | 正数から負数を減算できる | 1 - (-1)      | 2        |
| 減算       | 正数同士の減算ができる   | 1 - 1         | 0        |
| 減算       | 0を含む減算ができる      | 0 - 1         | -1       |
| 減算       | 最大値を含む減算ができる | Max - 1       | Max - 1  |
| 減算       | 最小値を含む減算ができる | -Max - 1      | Overflow |
| 減算       | 最大値同士の減算ができる | Max - Max     | 0        |
| 減算       | 最小値同士の減算ができる | -Max - (-Max) | 0        |

#### テストケース作成手順

上記の[レビューを実施](#レビューを実施)と同様の手順で、テストケース表を埋めた変更をしてgitでコミットしてください。

1. リポジトリのMarkdownファイルを開く(`docs/2-tutorial-sprint.md`)
1. テストケース表にテストケースを記載

    ```markdown title="docs/2-tutorial-sprint.md"
    | テスト観点 | テストケース             | 入力   | 期待値 |
    | ---------- | ------------------------ | ------ | ------ |
    | 減算       | 負数から正数を減算できる | -1 - 1 | -2     |
    | 減算       | 正数同士の減算ができる   | 1 - 1  | 0      |
    ```

1. 変更をコミット

    ```bash title="コミット例"
    git add docs/2-tutorial-sprint.md
    git commit -m "docs: チュートリアルスプリントのテストケースを追加"

    ```

1. 変更をプッシュ

    ```bash
    git push origin <your-branch-name>
    ```

## 探索的テスト

デプロイしたアプリを実際に動かし、挙動に問題がないか確認する。

確認観点は以下とする。

- 入力が正しく扱われるか
- 演算の分類が正しく動作するか
- 想定外の入力でエラーにならないか
- 表示内容や操作感に問題がないか
