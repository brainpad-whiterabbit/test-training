# チュートリアルスプリント

## スプリント概要

入力、演算の分類の機能を追加する。  
現状、足し算の機能しかないため、残りの機能を開発することを目的とする。

## 全体の流れ

1. 要件レビュー
2. テスト設計
3. 実装
4. テスト実装
5. 探索的テスト

## 要件レビュー

要件定義書を確認する。

対象は、要件定義書の「対応スプリント」列が「チュートリアルスプリント」になっている行とする。

レビュー対象は以下とする。

- 機能要件
- 非機能要件
- 受入条件

レビュー観点は以下とする。

- 要件内容に曖昧さがないか
- 理解できる文章になっているか
- 間違った記載がないか

## テスト設計

要件レビューの内容に基づき、テストケースを作成する。

## 実装

実装手順は後で記載する。

以下の手順も記載する。

- Gitのブランチを作成する
- 実装する
- commitする
- pushする
- CI/CDを実行する
- CI/CDの結果を確認する

## テスト実装

具体的なテスト作成手順は後で記載する。

テスト設計に基づき、単体テストを作成する。

また、要件定義書の受入条件に基づき、受入テストを作成する。

作成後、CI/CDでテスト結果を確認する。

## 探索的テスト

デプロイしたアプリを実際に動かし、挙動に問題がないか確認する。

確認観点は以下とする。

- 入力が正しく扱われるか
- 演算の分類が正しく動作するか
- 想定外の入力でエラーにならないか
- 表示内容や操作感に問題がないか
