# チュートリアルスプリント

## スプリント概要

現状、加算の機能のみ実装されています。このスプリントでは、減算の機能を追加することを目指します。

## 全体の流れ

再掲ですが、スプリント内で実施する内容は以下の通りです。

| スプリント期間 | 開発工程     | テスト工程                   |
| -------------- | ------------ | ---------------------------- |
| 前半           | ~~要件定義~~ | 要件レビュー, ~~テスト分析~~ |
| 前半〜中盤     | ~~設計~~     | ~~設計レビュー~~, テスト設計 |
| 中盤           | 実装         | テスト実装                   |
| 後半           | ~~テスト~~   | 探索的テスト                 |

以下の順番で実施してもらいます。

1. 要件レビュー
1. テスト設計
1. 実装
1. テスト実装
1. 探索的テスト

## 要件レビュー

今回は単体の機能を追加を目指す1スプリント内の要件レビューなため、本演習では以下と簡易的に定義します。

「スプリントで追加する機能の内容について、実現可能性、完全性、一貫性、明確性、テスト可能性などを確認する活動」

具体的に、追加機能の要件の記述に曖昧さや矛盾、不足がないか等の確認をしてもらいます。

### レビュー対象を確認

まず、[要件定義書](97-requirements.md)に記載されている内容を確認してください。機能要件、非機能要件に具体的な要件が記載されて定義されています。

- [非機能要件](97-requirements/#5-非機能要件)

各項目の表のうち「対応スプリント」列の記述がそのスプリントで追加する機能に対応する要件です。今回は「チュートリアルスプリント」と記載された行が対象になります。

### 関連するテスト観点を抽出

### テストケース作成

抽出したテスト観点を踏まえてテストケースを作成してください。以下の表を埋める形でもよいですし、形式を好みに合わせて変えてもよいです。

| テスト観点 | テストケース | 入力 | 期待値 |
| ---------- | ------------ | ---- | ------ |
|減算        |  正数同士の減算ができる           |      |        |
|            |              |      |        |
|            |              |      |        |

#### テストケース作成例

参考にテストケースの作成例 (TV-015 減算)を示します。

- 入力値を同値分割
    - 最小値, 負数, 0, 正数, 最大値
- 右と左の入力値の組み合わせを同値分割
    - (負 - 負)
    - (負 - 正)
    - (正 - 負)
    - (正 - 正)
    - 0を含む
    - 最大値を含む
    - 最小値を含む
    - (最大値 - 最大値)
    - (最小値 - 最小値)

テストケースに落とし込むと以下のようになります。

| テスト観点 | テストケース             | 入力                | 期待値   |
| ---------- | ------------------------ | ------------------- | -------- |
| 減算       | 負数同士の減算ができる   | -1 - (-3)           | 2        |
| 減算       | 負数から正数を減算できる | -1 - 1              | -2       |
| 減算       | 正数から負数を減算できる | 1 - (-1)            | 2        |
| 減算       | 正数同士の減算ができる   | 1 - 4               | -3       |
| 減算       | 0を含む減算ができる      | 0 - 1               | -1       |
| 減算       | 最大値を含む減算ができる | 999999 - 1          | 999998   |
| 減算       | 最小値を含む減算ができる | -999999 - 1         | Overflow |
| 減算       | 最大値同士の減算ができる | 999999 - 999999     | 0        |
| 減算       | 最小値同士の減算ができる | -999999 - (-999999) | 0        |

## 実装, テスト実装

要件定義書の内容を踏まえて、減算の機能の実装とそのテストを実装してください。

すでに加算の機能を実装しているため、参考にして減算の機能も同様の実装してください。

1. 実装
    - `src/training/app.py`
        - 電卓アプリのUIを実装している箇所
        - 減算のUIの挙動の実装が必要
    - `src/training/calculator.py`
        - 電卓の演算ロジックを実装している箇所
        - 減算のロジックの実装が必要

1. テスト実装
    - `tests/test_app.py`
        - 電卓アプリのUIのテストコードを実装している箇所
        - 減算のUIの挙動に対するテスト実装が必要
    - `tests/test_calculator.py`
        - 電卓の演算ロジックのテストコードを実装している箇所
        - 減算のロジックに対するテスト実装が必要

1. テスト実行
   全てのテストが通ることを確認してください。

    ```bash
    PYTHONPATH=src uv run pytest
    ```

1. 静的解析
    - 静的解析用のスクリプトを実行してエラーや警告が出ないか確認してください。

    ```bash
    ./scripts/check.sh
    ```

1. 変更をコミット

    ```bash title="コミット例"
    git add src/training/app.py src/training/calculator.py
    git commit -m "feat: 減算の機能を実装"

    ```

1. 変更をプッシュ

    ```bash
    git push origin <your-branch-name>
    ```

## 探索的テスト

アプリを実際に動かし、テストケースだけでは見つけにくい問題がないか確認します。

探索的テストでは、事前に作成したテストケースを順番に消化するだけでなく、操作しながら気になった点を深掘りします。今回のスプリントでは、追加した減算機能を中心に、既存の入力機能や画面表示との組み合わせも確認してください。

1. アプリを起動

    ```bash
    ./scripts/run-calculator-app.sh
    ```

1. 探索的テストを実施
1. 問題が見つかった場合は、修正して再度確認
1. 変更をコミット

    ```bash title="コミット例"
    git add src/training/app.py src/training/calculator.py
    git commit -m "fix: 減算の機能を修正"

    ```

1. 変更をプッシュ

    ```bash
    git push origin <your-branch-name>
    ```
