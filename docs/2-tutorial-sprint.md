# チュートリアルスプリント

## スプリント概要

現状、足し算の機能のみ実装されています。四則演算の残りの機能についてこのスプリントで実装してもらいます。

具体的に

- 引き算
- 掛け算
- 割り算

の機能を追加することになります。

## 全体の流れ

再掲ですが、スプリント内で実施する内容は以下の通りです。

| スプリント期間 | 開発工程     | テスト工程                   |
| -------------- | ------------ | ---------------------------- |
| 前半           | ~~要件定義~~ | 要件レビュー, ~~テスト分析~~ |
| 前半〜中盤     | ~~設計~~     | ~~設計レビュー~~, テスト設計 |
| 中盤           | 実装         | テスト実装                   |
| 後半           | ~~テスト~~   | 探索的テスト                 |

以下の順番で実施してもらいます。

1. 要件レビュー
1. テスト設計
1. 実装
1. テスト実装
1. 探索的テスト

## 事前準備

自分専用のgitブランチをmasterから切って作成してください。(作成済みならスキップ)

ブランチ名は命名規則 `ensyu/<year>/<lastname>.<firstname>`に従ってください。

```bash
git checkout -b ensyu/2026/taro.yamada
```

## 要件レビュー

一般的に要件レビューとは以下のような定義です。

「要件定義の内容について、プロジェクト目標との整合性、実現可能性、完全性、一貫性、明確性、テスト可能性などを確認する活動」

ただ、今回は単体の機能を追加を目指す1スプリント内の要件レビューなため、本演習では以下と簡易的に定義します。

「スプリントで追加する機能の内容について、実現可能性、完全性、一貫性、明確性、テスト可能性などを確認する活動」

具体的に、追加機能の要件の記述に曖昧さや矛盾、不足がないか等の確認と、受入条件との整合性およびテスト可能性を確認してもらいます。

要件レビューを実施することで、定義間違いによる手戻りの防止や開発効率化、品質向上が期待できます。

### 作業の流れ

以下の手順に従ってください。

1. レビュー対象を確認
1. レビュー観点を確認
1. レビューを実施

### レビュー対象を確認

まず、[要件定義書](97-requirements.md)に記載されている内容を確認してください。機能要件、非機能要件、受入条件に具体的な要件が記載されて定義されています。

- [機能要件](97-requirements/#3)
- [非機能要件](97-requirements/#4)
- [受入条件](97-requirements/#5)

各項目の表のうち「対応スプリント」列が「チュートリアルスプリント」となっている行の記述がこのスプリントで追加する機能に対応する要件です。

### レビュー観点を確認

以下をレビュー観点とします。

- 曖昧な表現や誤りが含まれていないか
    - 曖昧な例: 「適切に」, 「簡単に」など
- 要件内容について、主体、入力、処理、出力、条件が明確に理解できるように記載されているか
    - 理解できれば必ずしも含めなくてもよい
- 第三者でも理解できて簡潔な文章になっているか
- 記載粒度が統一されているか
- 用語の使い方が統一されているか
- 未記載・記載漏れがないか
- 要件同士で矛盾がないか
- テスト可能か

### レビューを実施

レビュー観点に従って対象の項目をレビューして、もし修正点があれば、レビュー点の列に修正内容を記載してください。

1. リポジトリのMarkdownファイルを開く(`docs/97-requirements.md`)
1. レビュー内容を記載
    - 修正点があれば修正を反映して、メモ程度でよいのでレビュー点を記載

    ```markdown title="docs/97-requirements.md"
    |    No | 分類 | 機能名   | 要件内容                       | レビュー点             | 対応スプリント           |
    | ----: | ---- | -------- | ------------------------------ | ---------------------- | ------------------------ |
    | F-006 | 演算 | 四則演算 | 加算、減算、乗算、除算ができる | {ここにレビュ点を記載} | チュートリアルスプリント |
    ```

1. 変更をコミット

    ```bash title="コミット例"
    git add docs/97-requirements.md
    git commit -m "docs: チュートリアルスプリントの要件レビュー内容を反映"

    ```

1. 変更をプッシュ

    ```bash
    git push origin <your-branch-name>
    ```

## 探索的テスト

デプロイしたアプリを実際に動かし、挙動に問題がないか確認する。

確認観点は以下とする。

- 入力が正しく扱われるか
- 演算の分類が正しく動作するか
- 想定外の入力でエラーにならないか
- 表示内容や操作感に問題がないか
